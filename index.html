Инициализация Git и отправка на GitHub

git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin git@github.com:Rusificator/web-hw2.git
git push -u origin main
Git push
Локальный репозиторий создан, все файлы добавлены, создан коммит. После связывания с удалённым репозиторием изменения успешно отправлены на GitHub.

Клонирование репозитория на сервер

git clone git@github.com:Rusificator/web-hw2.git
mkdir -p ~/www/hw2
cp -r ~/web-hw2/* ~/www/hw2/
Клонирование на сервере
Репозиторий склонирован в домашнюю директорию, создан каталог ~/www/hw2, файлы скопированы в веб-доступное место.

Подготовка HTTP-запросов и настройка PuTTY

Тексты запросов
Все запросы для пунктов 1–7 были заранее набраны в текстовом редакторе для удобного копирования в PuTTY.
Настройки PuTTY
Для отправки запросов используется PuTTY в режиме Raw (прямое TCP-соединение). Адрес: u82457.kubsu-dev.ru, порт 80.
Окно PuTTY ожидает запрос
После открытия соединения появляется пустое окно, готовое для вставки HTTP-запросов (правая кнопка мыши).

2. Выполнение HTTP-запросов

Пункт 1. Получить главную страницу методом GET в протоколе HTTP/1.0 GET / HTTP/1.0
GET / HTTP/1.0
Host: u82457.kubsu-dev.ru

Статус: HTTP/1.1 200 OK – запрос выполнен успешно.
Заголовки: Server: Apache/2.4.58 (Ubuntu), Content-Length: 10671, Content-Type: text/html; charset=UTF-8.
Тело ответа содержит стандартную страницу-заглушку Apache2 Ubuntu Default Page.
Пункт 2. Получить внутреннюю страницу методом GET в протоколе HTTP/1.1 GET /hw2/index.php
GET /hw2/index.php HTTP/1.1
Host: u82457.kubsu-dev.ru
Connection: close

Статус: HTTP/1.1 404 Not Found – задан самим скриптом.
Заголовок Content-Type: text/html; charset=UTF-8.
Тело ответа содержит вывод скрипта: Array ( ) Привет, мир! (пустой массив $_POST и строка).
Пункт 3. Определить размер файла file.tar.gz, не скачивая его HEAD /hw2/file.tar.gz
HEAD /hw2/file.tar.gz HTTP/1.1
Host: u82457.kubsu-dev.ru
Connection: close

Статус: HTTP/1.1 200 OK – файл существует.
Заголовок Content-Length: 11335 – размер файла в байтах.
Тип файла: Content-Type: application/x-gzip.
Пункт 4. Определить медиатип ресурса /image.png HEAD /hw2/image.png
HEAD /hw2/image.png HTTP/1.1
Host: u82457.kubsu-dev.ru
Connection: close

Статус: HTTP/1.1 200 OK.
Заголовок Content-Type: image/png – медиатип PNG.
Размер: Content-Length: 10506 байт.
Пункт 5. Отправить комментарий на сервер по адресу /index.php (POST) POST /hw2/index.php
POST /hw2/index.php HTTP/1.1
Host: u82457.kubsu-dev.ru
Content-Type: application/x-www-form-urlencoded
Content-Length: 12
Connection: close

comment=Hello
Статус: HTTP/1.1 404 Not Found (задан скриптом).
Заголовок Content-Type: text/html; charset=UTF-8.
Тело ответа: Array ( [comment] => Hello ) Привет, мир! – значение поля comment успешно передано.
Пункт 6. Получить первые 100 байт файла /file.tar.gz Range: bytes=0-99
GET /hw2/file.tar.gz HTTP/1.1
Host: u82457.kubsu-dev.ru
Range: bytes=0-99
Connection: close

Статус: HTTP/1.1 206 Partial Content – частичное содержимое.
Заголовок Content-Range: bytes 0-99/11335.
Content-Length: 100 – длина переданных данных.
Тело содержит первые 100 байт файла (бинарные данные, отображаемые как нечитаемые символы).
Пункт 7. Определить кодировку ресурса /index.php HEAD /hw2/index.php
HEAD /hw2/index.php HTTP/1.1
Host: u82457.kubsu-dev.ru
Connection: close

Статус: HTTP/1.1 404 Not Found (задан скриптом).
Заголовок Content-Type: text/html; charset=UTF-8 – кодировка UTF-8 указана явно.
